# Claim Boundary

**admissibility-kernel v0.1.0 — Public Inspection Release**  
**Mebs Loghdey / Neverthought**

This repository is a public learning and inspection surface for **context-to-consequence pre-commitment admissibility governance**.

It makes one class of question inspectable:

> **Can this proposed commitment proceed without destroying the system’s future capacity for legitimate transformation?**

It does not claim to answer every question about safety, truth, legitimacy, institutional authority, legality, or production deployment. Humanity has enough systems that pretend to do everything. This one does not.

---

## What this repository claims

This repository claims that it provides:

1. A public-safe executable implementation of a pre-commitment admissibility pattern.
2. A four-outcome governance taxonomy: `EXECUTE`, `TRANSFORM`, `DEFER`, `REJECT`.
3. A public schema for inspection records associated with those outcomes.
4. A demonstrable separation between current state, projected state, and observed state.
5. A public learning pathway for understanding context-to-consequence governance.
6. A conceptual bridge between human-before-the-loop judgement and governed execution.

---

## What this repository does not claim

This repository does **not** claim to:

- certify AI systems;
- determine legal or regulatory compliance;
- prove safety;
- determine institutional legitimacy;
- replace human judgement;
- provide production deployment governance;
- expose Neverthought’s full internal kernel;
- expose calibrated thresholds, commercial adapters, bench internals, or proof-generation mechanics;
- provide a complete implementation of Epistemic Integrity Governance or the Semantic Ruliad;
- evaluate truth, morality, or organisational readiness in general.

---

## Inspection standard

Every artefact should be interpreted through this public standard:

```text
claim → evidence object → inspection path → claim limit
```

A local test passing proves only that the local public implementation behaves as specified by its tests. It does not certify real-world behaviour.

An example producing `EXECUTE` proves only that the example crossed the public demonstration gate. It does not authorise any real deployment.

A decision record is an inspection object. It is not a legal conclusion, regulatory conclusion, or warranty.

---

## Public threshold status

Thresholds, floors, adjustments, and routing heuristics in this repository are **illustrative public proxy values**.

They are included to make the architecture runnable and inspectable. They are not calibrated deployment values.

Calibrated runtime components, domain-specific thresholds, evidence contracts, proof chains, certification envelopes, and private adapters are outside this public release.

---

## Relationship to the website and bench

The public website explains the doctrine.

The bench teaches and simulates judgement.

This repository exposes the minimal executable inspection surface.

Together, they form a self-service build-and-learn path:

```text
learn the question → run the kernel → inspect the record → understand the boundary → build with discipline
```
