## Development Timeline

The Epistemic Integrity Governance (EIG) framework was initiated, establishing
the core invariant kernel ⟨A, I, μ, ∂D⟩ and the foundational claim that
admissibility is the correct governance locus — not output correctness, policy
compliance, or task completion.

### 2025 — Formalisation and Canon Development

The Semantic Ruliad canon was developed across 20+ versioned layers, including:

- Foundational Technical Brief (v3.0)

- Execution Specification (v1.0)
- Calculus of Admissible Commitment (v0.1)
- NSV (Negative Space Vector) formalisation
- Ten-law machinery specification

Key concepts formalised: pre-commitment as the governance locus, admissibility
predicate with six axioms, OstinatoLock failure mode, artisanal inference as
practitioner framing, ECL (Epistemic Commitment Loss) classes.

### 2026 Q1 — Named Asset Specification

Three technical assets named and specified as distinct IP:

- **Pre-Commitment Admissibility Gate (PCAG)** — three-state separation formalised
- **Recursive Viability Kernel (RVK)** — seven recursion failure modes named
- **Transformability Preservation Invariant (TPI)** — governing invariant formalised

The DEFER non-transition record and TRANSFORM constrained reframing operator
defined. JSON schemas produced for AdmissibilityDecisionRecord,
RecursiveViabilityRecord, and TransformabilityPreservationRecord.
