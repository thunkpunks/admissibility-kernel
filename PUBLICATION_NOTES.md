# Publication Notes

**admissibility-kernel v0.1.0 — Public Inspection Release**

This repository is intended to be Neverthought’s first public code artefact.

It should set the tone for the public technical estate going forward:

- commercially serious;
- pedagogically centred;
- claim-bounded;
- self-service friendly;
- inspectable by developers, researchers, buyers, and deployment practitioners;
- protective of the deeper system.

The repo should feel like the first public instrument in a larger institution, not a weekend experiment with unusually dramatic nouns.

---

## Public role

This repo is the first public instrument in the **context-to-consequence stack**.

It is designed to work alongside:

- the Neverthought website;
- the self-service bench;
- public learning modules;
- future deployment judgement curricula;
- future commercial diagnostic, training, and evidence workflows.

## What to pin publicly

Use these phrases consistently across GitHub, the website, and launch posts:

- context-to-consequence governance
- human-before-the-loop
- pre-commitment admissibility
- consequence-aware commitment
- transformability preservation
- recursive viability
