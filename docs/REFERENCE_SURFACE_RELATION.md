# Relation to Reference-Surface Validation

Reference-surface validation and pre-commitment admissibility solve adjacent but different problems.

## Reference-surface validation asks

> Is the governing surface structurally eligible to be used for inspection?

That is a surface-level eligibility question. It helps determine whether a reference structure is internally coherent enough to govern a later judgement.

## Pre-commitment admissibility asks

> Should this proposed action be allowed to cross a commitment boundary under recursive pressure?

That is an action-governance question. It treats reference surfaces, evidence, proposal structure, recoverable optionality, topology temperature, and residue as inputs to a commitment decision.

## Layer distinction

| Layer | Question | Typical output |
|---|---|---|
| Reference-surface validation | Is the governing surface structurally eligible? | pass / fail / receipt |
| Pre-commitment admissibility | Can this proposed action execute without destroying future legitimate transformation? | execute / transform / defer / reject |
| Recursive viability | Can repeated local passes remain globally governable? | viable / conditional / defer / non-viable |
| Transformability preservation | Can the system still become otherwise? | preserved / contracted / transform / defer / violated |

The public positioning is therefore complementary, not derivative: reference-surface validation may validate a governing surface; admissibility decides whether a commitment is allowed to proceed.
