# Adjacent Work Boundary

This repository is adjacent to execution-boundary governance, reference-surface validation, audit receipts, and runtime control systems. It is not reducible to any of them.

---

## Execution-boundary governance

Execution-boundary governance asks:

> Where does the system physically stop?

That work is concerned with whether a system can prevent mutation, execution, escalation, or side effects when required proof is missing.

The admissibility kernel asks an earlier and more general question:

> Should the proposed commitment be allowed to cross the boundary at all?

---

## Reference-surface validation

Reference-surface validation asks:

> Is the governed surface externally grounded, current, falsifiable, and institutionally legitimate?

This is important. A gate can correctly enforce the wrong surface. Humanity has built whole bureaucracies on that thrilling premise.

The admissibility kernel treats surfaces as inputs to commitment governance. A surface may be valid and still produce an inadmissible commitment under pressure.

---

## Neverthought’s public territory

Neverthought’s public territory here is:

```text
context-to-consequence pre-commitment admissibility governance
```

The distinctive question is:

> Can this proposed commitment proceed without destroying the system’s future capacity for legitimate transformation?

The distinctive output is:

```text
EXECUTE | TRANSFORM | DEFER | REJECT
```

The distinctive public evidence object is:

```text
AdmissibilityDecisionRecord
```
