# Website Bridge

This repository is designed to connect to Neverthought’s public self-service build-and-learn environment.

The website explains the doctrine.  
The bench demonstrates judgement.  
The repo exposes a minimal executable inspection surface.

Together, they form a public learning route:

```text
Orient → Observe → Run → Inspect → Reflect → Build
```

---

## Public user journey

A serious visitor should be able to move through the ecosystem like this:

```text
1. Learn the question.
   What must remain possible after the system acts?

2. See the pipeline.
   Context → Consequence → Admissibility → Commitment.

3. Run the kernel.
   Try a proposed commitment and inspect the outcome.

4. Read the record.
   Understand what was preserved, contracted, deferred, or refused.

5. Build with discipline.
   Use the grammar to design better AI-supported workflows, demos, training, or governance artefacts.
```

---

## What belongs on the website

The website should explain:

- the governing question;
- context-to-consequence;
- human-before-the-loop;
- the four outcomes;
- why refusal and deferral are valuable;
- how the bench keeps asking and answering these questions through public-safe simulations.

The website should not disclose calibrated internals, private thresholds, domain-specific adapters, or commercial certification mechanics.

---

## What belongs in the repo

The repo should expose:

- public proxy code;
- schemas;
- examples;
- tests;
- claim boundaries;
- learning docs;
- inspection record grammar.

The repo should not expose the production kernel, private bench internals, calibrated runtime values, deployment envelopes, or commercial adapters.
