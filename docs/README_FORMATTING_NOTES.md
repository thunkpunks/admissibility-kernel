# README Formatting Notes

This repository uses a public-inspection README pattern:

```text
identity line -> boundary -> inspection standard -> core question -> evidence surfaces -> claim ceiling
```

The purpose is to make the repository searchable, readable, and defensible without over-disclosing protected architecture.

## Required top-level pattern

1. One-line identity route.
2. One sentence explaining where governance becomes real.
3. Public inspection boundary.
4. Public inspection standard.
5. Search identity and core terms.
6. Governing question.
7. Repository-specific proof surfaces.
8. Claim boundary and claim ceiling.

## Public-safe wording rule

Do not say:

```text
This is the kernel.
```

Say:

```text
This is a public inspection layer for the admissibility architecture.
```

Do not say:

```text
Certified / production-safe / compliant / complete.
```

Say:

```text
Bounded public artefact with explicit claim limits.
```
