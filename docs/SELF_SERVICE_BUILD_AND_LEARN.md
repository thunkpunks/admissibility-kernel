# Self-Service Build and Learn Ethos

This repository is built for people who need to understand, test, and build consequence-aware governance patterns without waiting for a priesthood to descend from a slide deck.

The ethos is:

```text
teach the question → expose the artefact → run the example → inspect the record → build more carefully
```

---

## Pedagogical centre

The repo is not just a code sample. It is a learning instrument.

It should help developers, researchers, deployment leads, FDEs, governance practitioners, and serious buyers learn a professional grammar:

- action is not the same as commitment;
- context is not the same as consequence;
- confidence is not admissibility;
- compliance is not transformability;
- refusal can be a valid system behaviour;
- deferral can preserve future legitimacy;
- transformation can be more intelligent than execution.

---

## Commercial seriousness

The public surface is deliberately bounded because serious buyers do not need theatrical omniscience. They need discipline:

- clear scope;
- visible schemas;
- executable examples;
- testable claims;
- inspection records;
- documented limits;
- no inflated readiness, safety, or compliance claims.

This is how a public learning repo becomes a commercial trust surface without accidentally becoming an IP donation box.

---

## Build posture

The public repo should invite people to build with the grammar, not clone the internal engine.

Acceptable public build activity:

- educational demos;
- toy adapters;
- inspection-record experiments;
- curriculum modules;
- scenario fixtures;
- non-production governance simulations.

Not exposed here:

- calibrated thresholds;
- domain adapters;
- private bench signals;
- production evidence contracts;
- deployment certification envelopes;
- commercial runtime policy machinery.
